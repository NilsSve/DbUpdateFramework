Bugs Fixed in Version 1.0.0.16 (Released June 30 2017)

    • When using the SqlDatabaseCreate message the logic would fail and the program would exit. In fact the database was created just fine. Instead a recent change to the SqlUtilExecuteQuery message that traversed the response from the ESQL call could in some special circumstances (and this was one of them) make changes to the system variables Err and LastErr. However, the logic was supposed to clean up after itself but it did a poor job in that only the Err flag was reset – not the LastErr. The SqlDatabaseCreate checked the LastErr flag and therefore the rest of the update logic was interrupted and the program exited.

Reported by Sam Neuner


    • It turned out there was another bug related to the SqlUtilExecuteQuery which resulted in that the DAW class cCLIHandler wasn't properly updated after changing the database.
    • A bug related to drag & drop of files was discovered and fixed in the sample program DbUpdateFrameworkTests.exe. After adding scrolling containers drag & drop of a Filelist.cfg to the “UtilFunctions” tab-page didn't work.
